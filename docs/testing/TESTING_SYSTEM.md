# 测试体系说明

## 测试目标

本项目重点展示以下测试能力：

- 功能测试
- 边界测试
- 异常测试
- 压力测试
- 自动化回归

## 已覆盖方向

主要覆盖：

- 角色控制与 Jump 兼容性
- 战斗、掉落、拾取、背包联动
- 场景边界与高低差交互
- AI Stress Spawn 梯度压测
- 自动化记录与性能报告输出

## 测试分层

### 手工功能测试

用于验证核心玩法链路是否正确，包括：

- 角色移动、攻击、Jump
- 背包增删改、装备、读取
- 掉落生成和拾取
- 边界与地形交互

### 压力与性能测试

用于验证：

- AI 数量增长后的性能变化
- 生成范围变化后的行为和性能差异
- 高密度战斗和掉落堆积场景下的稳定性

### 自动化回归

当前自动化的重点不是复杂行为回放，而是：

- 自动启动测试会话
- 自动收集性能数据
- 自动输出 `csv / summary / autotest`
- 自动做轻量阈值判定

## 当前测试价值

这套体系的重点不是“覆盖所有测试类型”，而是把一个现有 UE 样例项目扩展成：

- 具备清晰测试思路
- 有真实缺陷与风险记录
- 有性能趋势结论
- 有可重复自动化产物

的完整展示项目。
