# 缺陷与风险分析

## 已确认问题

### 1. AI 之间可能互相伤害

现象：

- 多 AI 战斗场景下，敌人会对其他敌人造成伤害

影响：

- 破坏战斗规则
- 在高密度 AI 场景下污染测试结果
- 放大压力测试中的对象流转和掉落叠加

根因判断：

- 项目里存在阵营概念，但没有收口到伤害目标过滤链

### 2. 目标选择与效果应用之间缺少统一过滤

影响：

- 各技能可能在友军过滤、有效目标判断上表现不一致
- 规则更容易散落到蓝图层

### 3. 装备槽写入缺少合法性校验

风险：

- 可能出现非法装备状态
- 存档恢复时可能形成“未持有但已装备”的脏状态

## 结构性风险

### 4. 蓝图耦合度高

表现为：

- 玩法入口更多掌握在蓝图层
- AutoPlay 等入口缺少稳定 C++ API
- 功能扩展和自动化接入成本偏高

### 5. 动画与资源替换耦合较强

表现为：

- Skeleton、Animation Blueprint、Montage 与角色资源绑定明显
- 外部角色替换更像兼容性实验，而不是低成本接入流程

## 总体判断

该项目不是“完全不可维护”的黑盒，但存在明显的样例工程特征：

- C++ 有骨架
- 蓝图承担大量最终玩法实现
- 规则收口不足
- 适合做灰盒测试、结构分析和风险识别
