# 缺陷与风险记录

## 文档说明

本文件记录在 UE5 ActionRPG Demo 测试与功能扩展过程中观察到的问题、设计限制与潜在风险。

由于该项目为开源 Demo，本报告不以商业项目正式缺陷单为标准，而主要用于记录：

- 测试过程中发现的问题
- 系统设计上的可用性问题
- 功能扩展后的兼容性风险
- 项目结构与维护风险

---

# P1 玩法与战斗相关问题

## 1. Jump 扩展后出现高地规避 AI 问题

**类型：** 玩法问题 / 功能扩展兼容性问题  

**现象：**

扩展 Jump 功能后，玩家可跳至高处地形，而 AI 无法追击至该位置。

表现为：

- AI 停止追击
- AI 无法攻击玩家
- 玩家可在高处持续输出

**影响：**

破坏战斗平衡，玩家可通过地形规避敌人攻击。

**可能原因：**

- NavMesh 未覆盖高处区域
- AI 寻路逻辑未处理高低差
- 原项目默认假设角色在同一平面移动

**建议：**

- 检查 NavMesh 覆盖范围
- 为高差地形设计 AI 可达性逻辑
- Jump 扩展时同步调整 AI 导航系统

# P2 场景与物理问题

## 2. 敌人贴近障碍物击杀后出现飞起现象

**类型：** 场景碰撞问题  

**现象：**

当敌人贴近石头或其他场景障碍物时被击杀，角色可能出现向上或向外弹起的异常行为。

**复现步骤：**

1. 将敌人引至石头或障碍物附近
2. 在贴近障碍物状态下击杀敌人
3. 观察敌人死亡表现

**实际结果：**

敌人死亡瞬间出现异常弹起。

**可能原因：**

- 角色 Capsule 与场景碰撞体重叠
- 死亡动画触发物理状态变化
- Root Motion 与碰撞体产生冲突

**建议：**

- 优化死亡动画与物理状态切换
- 检查角色 Capsule 与场景碰撞设置

---

# P3 交互与可用性问题

## 3. 武器与道具切换流程不够友好

**类型：** 交互设计问题  

**现象：**

玩家在切换武器或使用药品时，需要进入背包界面手动操作。

**影响：**

战斗过程中操作效率较低，影响战斗节奏。

**建议：**

- 增加快捷装备栏
- 支持战斗中快速切换武器或道具
- 减少频繁进入背包界面

---

# P4 架构与维护风险

## 4. 蓝图耦合度较高，扩展成本较大

**类型：** 架构风险 / 可维护性问题  

**现象：**

在尝试扩展功能与替换角色资源过程中发现：

- Animation Blueprint 与 Skeleton 强耦合
- Animation Montage 与攻击逻辑绑定
- 多个蓝图之间依赖关系复杂

**影响：**

功能扩展、资源替换与系统维护成本较高。

**建议：**

- 降低动画系统与角色资源的直接耦合
- 将战斗逻辑与表现层解耦
- 建立角色资源适配流程

---

# P5 Demo功能限制

## 5. 部分完整游戏流程功能缺失

**类型：** 功能缺失（Demo 范围）  

当前项目未提供以下系统：

- 存档 / 读档系统
- 分数或战斗统计系统

该问题属于 Demo 项目范围限制，对核心玩法演示影响较小。

---

# 当前测试结论

在已覆盖的核心玩法场景中，项目整体运行稳定，未发现阻断级功能缺陷。

目前观察到的问题主要集中在：

- Jump 扩展后暴露的 Z 轴交互问题
- 场景碰撞导致的异常物理行为
- 部分交互体验问题
- 蓝图结构耦合带来的维护风险
