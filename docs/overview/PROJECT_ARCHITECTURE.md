# 项目架构说明

## 项目概述

该项目基于 UE5 ActionRPG 样例工程展开，整体形态不是纯 C++ 项目，也不是纯蓝图项目，而是典型的 Unreal 混合架构：

- UE 提供运行时框架、对象系统、资源系统、输入系统、GAS、AI 和编辑器能力
- C++ 负责系统骨架、数据结构、存档、物品和能力系统基础实现
- 蓝图负责玩法编排、表现层、UI、具体 AI 行为和资源装配

## 三层关系

### UE 层

UE 层负责：

- 对象生命周期、反射、GC、序列化
- 输入、动画、导航、AI、资源加载
- Gameplay Ability System
- 编辑器和内容生产流程

### C++ 层

C++ 层负责：

- 角色、控制器、GameInstance、AssetManager 等系统基座
- 背包、装备槽、存档、能力、属性等规则层接口
- 对蓝图开放可调用能力和事件入口
- 为测试和观测能力提供可扩展接口

### 蓝图层

蓝图层负责：

- 玩家和敌人的具体玩法逻辑
- 动画、Montage、特效、UI、行为树、输入资源
- AutoPlay、AI 行为入口和场景交互表现

## 当前项目特征

该项目的工程特征可以概括为：

- C++ 有较完整的系统骨架
- 蓝图承担了大量最终玩法实现
- 玩法入口和资源绑定关系更多掌握在蓝图层
- 这使它非常适合做系统测试、灰盒分析和扩展性评估

## 结构亮点

从展示仓库角度，这个项目的亮点不在于“重写了整个游戏框架”，而在于：

- 能基于现有 UE 样例工程梳理系统边界
- 能识别蓝图高耦合对测试与扩展的影响
- 能在不大改蓝图的情况下，从 C++ 层补足可观测性和自动化能力
