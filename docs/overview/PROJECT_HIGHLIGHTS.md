# 项目亮点

## 亮点一：基于现成 ARPG 样例做系统化测试拆解

本项目不是简单体验 Demo，而是围绕现有 UE5 ActionRPG 样例做系统级拆解，覆盖：

- 角色控制
- 战斗与技能
- AI 行为
- 掉落与拾取
- 背包与装备
- 场景边界
- 性能与压力测试

## 亮点二：在现有玩法基础上扩展可测试场景

为支撑更有代表性的测试场景，我补充了：

- Jump 功能扩展
- AI Stress Spawn 场景构造
- 第三方角色资源替换实验

这些扩展不是为了“加玩法”，而是为了扩大测试覆盖面、暴露原始项目结构依赖和兼容性风险。

## 亮点三：从 C++ 层补足可观测性

在不修改蓝图资源的前提下，新增了：

- 性能监控子系统
- 自动化测试子系统
- 关键链路性能埋点
- 自动化报告输出

因此项目不再只是“手工测试 + 观察现象”，而是具备了：

- 自动记录
- 自动输出
- 可重复回归
- 可对比分析

## 亮点四：保留真实工程边界

本项目保留了原样例工程的真实限制，例如：

- AutoPlay 入口主要在蓝图层
- 复杂玩法逻辑与蓝图资源仍有较强耦合

展示重点不是掩盖这些边界，而是说明：

- 如何识别这些边界
- 如何在边界内做有效测试扩展
- 如何把已有项目打造成可展示的测开作品
