# UE5 ActionRPG Demo 游戏系统测试项目

## 项目简介

本项目基于开源 **UE5 ActionRPG Demo**，对其核心玩法系统进行系统化测试与分析。

测试内容覆盖：

- 角色控制系统
- 战斗系统
- AI 行为
- 掉落与拾取系统
- 背包与商店系统
- 游戏性能表现

同时对部分功能进行了实验性扩展（如 **Jump 功能启用** 与 **角色资源替换实验**），用于分析系统兼容性与潜在风险。

本仓库主要用于展示 **游戏测试流程、测试用例设计、缺陷分析与性能测试结果**。

---

# 测试目标

本项目通过系统性测试验证以下内容：

- 核心玩法系统的稳定性
- 不同系统之间的交互行为
- 战斗与 AI 系统在复杂场景下的表现
- 高密度 AI 场景下的性能表现
- UE 动画系统与角色资源之间的依赖关系

# 测试方法

测试用例设计采用以下软件测试方法：

- 等价类划分
- 边界值分析
- 场景法
- 灰盒测试  

灰盒测试主要通过：

- 调整 AI 刷新数量
- 修改蓝图参数
- 构造特殊场景

以验证系统在异常或边界条件下的行为表现。

---

# 核心测试场景

| 场景 | 覆盖系统 | 描述 |
|-----|------|------|
| 战斗 → 掉落 → 拾取 | 战斗 / 掉落 / 背包 | 验证击杀敌人后的掉落生成与背包更新 |
| 商店购买流程 | 商店 / 背包 / 数据存储 | 验证金币扣减、物品入包与数据保存 |
| Jump 功能兼容性测试 | 角色控制 / 战斗 | 验证新增 Jump 与攻击、落地逻辑的兼容性 |
| AI 寻路测试 | AI / 导航系统 | 验证玩家处于高地时 AI 的追击行为 |
| 角色资源替换实验 | 动画系统 / Skeleton | 分析动画蓝图与角色资源的依赖关系 |

# 性能测试

通过构建 **AI Stress Test 场景**进行压力测试。

AI 数量测试梯度：

`0 → 10 → 50 → 100 → 500 → 1000`

使用 UE 实时性能监控工具：

`stat fps\ stat unit\ stat unitgraph`

主要关注指标：

- FPS 变化
- CPU / GPU 帧时间
- Game Thread 开销
- AI Tick 开销
- Actor 数量变化

# 实验记录：角色资源替换

在实验过程中尝试将默认角色替换为第三方角色资源。

测试过程中观察到：

- Animation Blueprint 与 Skeleton 存在强依赖
- Animation Montage 与攻击逻辑存在绑定
- 直接替换角色资源可能导致攻击逻辑异常

该实验用于分析 **UE 动画系统的资源依赖关系与扩展风险**。
